# Contributing to AIDE

Thank you for your interest in improving AIDE! This framework is designed to evolve based on real-world usage.

## How to Contribute

### 🐛 Report Issues

Found a problem? Let us know:

1. Check [existing issues](https://github.com/ZaxShen/AIDE/issues) first
2. Create a new issue with:
   - Clear description of the problem
   - Steps to reproduce (if applicable)
   - What you expected vs. what happened
   - Which AI assistant you're using (Claude, GPT-4, etc.)

### 💡 Suggest Improvements

Have ideas for better workflows?

1. Open an issue with tag `enhancement`
2. Describe:
   - What problem it solves
   - How it would work
   - Why it's better than current approach
   - Any trade-offs

### 📝 Share Examples

Help others learn from your experience:

1. Fork the repo
2. Add your example to `examples/` directory:
   - `examples/your-project-name/TASKS.md`
   - `examples/your-project-name/logs/`
   - `examples/your-project-name/README.md` (brief description)
3. Submit a pull request

### 🔧 Code Contributions

Want to improve the framework itself?

1. Fork and clone the repo
2. Create a new branch: `git checkout -b feature/your-feature-name`
3. Make your changes
4. Test with at least one AI assistant
5. Update documentation if needed
6. Submit a pull request with:
   - Clear description of changes
   - Why it improves AIDE
   - Example of usage

## Guidelines

### Writing Style

- **Concise**: AIDE is "minimum best practice" - keep it minimal
- **Clear**: Avoid jargon, write for developers of all levels
- **Actionable**: Every instruction should be executable

### Philosophy

AIDE is built on these principles:

1. **Human Control**: Never sacrifice control for autonomy
2. **Sequential Execution**: One thing at a time, done right
3. **Complete Transparency**: Log everything for learning and audit

Contributions should strengthen these principles, not dilute them.

## Questions?

- Open a [discussion](https://github.com/ZaxShen/AIDE/discussions)
- Tag issues with `question`

## Code of Conduct

Be respectful, constructive, and helpful. We're all here to make AI collaboration better.
